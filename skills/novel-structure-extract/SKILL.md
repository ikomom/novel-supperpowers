# Novel Structure Extract

用这个 skill 把原文提炼成可复用的结构笔记。

## 目标

产出的不是泛泛总结，而是后面能直接拿来施工的结构材料。

如果原文很长，结构提炼这一步本身就应该被当成独立阶段，不要和后续设计或正文起稿混在一个动作里完成。

优先产出：
- arc 总结
- 人物功能笔记
- 章节功能清单
- 母题追踪
- 结构转折点
- 重写约束

## 核心规则

### 1. 没读够，不下结论

如果用户要的是整段理解，就不要只凭几个场景硬推全段结构。

### 1.5 长篇先做机器辅助索引，不要硬吃全文

如果原文是几百章、上千章、或单文件超长 txt，不要假设模型可以一次性可靠理解全书。

优先做两步：
- 先用脚本建立章节索引、标题表、范围预览
- 再只读取当前任务真正相关的章节范围

推荐先运行：`scripts/build_story_map.py`

这个脚本适合：
- 从超长 txt 中提取章节边界
- 生成章节标题、起始行、粗略预览
- 帮后续结构提炼只读取必要范围，而不是整本硬读

### 2. 区分“表层发生了什么”和“这一章真正干了什么”

每章至少识别两层：
- 表层事件
- 结构功能

常见结构功能包括：
- 扰动
- 升压
- 假性缓和
- 揭示
- 关系变化
- 主题翻面
- 余波

### 3. 追踪反复出现的装置

重点观察：
- 身份变化
- 镜像与二重身
- 重复物件
- 命名模式
- 梦/现实接口
- 反复出现的句式逻辑

### 4. 笔记必须服务后续使用

好的结构提炼应该回答：
- 哪些东西绝对不能丢
- 哪些东西可以压缩
- 这一段真正写的是什么
- 后续重写必须继承什么

## 推荐流程

1. 如果文本超长，先给出本轮读取计划
2. 如果文本超长，先跑章节索引脚本
3. 只读取目标章节范围
4. 先列一版逐章功能清单
5. 再提炼整段主脊梁
6. 再提炼人物功能和压力点
7. 补阶段状态、关系网和世界秩序
8. 最后写重写约束与压缩机会

如果当前任务还会继续进入“设计下一 arc”或“直接续写正文”，先把这一轮结构提炼结果单独写清，再进入下一步，不要一边提炼一边开写。

## 如果后续要做续写，结构提炼不能只停在剧情摘要

至少补出这三层：

### 1. 阶段状态
- 主角现在处于什么人生阶段
- 核心配角分别处于什么阶段
- 当前段落结束后，谁已经变了，谁还没变

### 2. 关系网
- 主角与关键人物的关系走到了哪一步
- 哪些关系已经完成阶段性兑现
- 哪些关系仍然有未兑现的压力

### 3. 世界秩序
- 当前世界稳定成立的规则是什么
- 最近的重大事件改写了什么格局
- 哪些变化已经开始反过来塑造人物生活

如果后续使用者需要“接着原作继续写”，这些信息必须在结构笔记里显式写出来，不要只让它隐含在总结里。
