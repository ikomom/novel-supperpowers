# 从上一篇挖下一篇

在发明“下一篇”之前，先把上一篇挖干净。

## 要挖什么

### 1. 后果

列出上一篇真正改变了什么：
- 身体或精神后遗症
- 信任关系变化
- 某个旧判断被打碎
- 某个地点或记录被毁坏/暴露
- 新暴露出来的敌人、结构或系统

### 2. 未完压力

找那些没有真正收完的东西：
- 关系只修了一半
- 某种机制只揭了一层
- 某句警告还没兑现
- 某个活下来的人还远没稳定
- 主角身上刚新长出来的脆弱点

### 3. 还在发力的母题

只有还带压力的母题才值得继续用。

例如：
- 名字
- 镜像
- 面具
- 睡与醒
- 档案与记录
- 二重身
- 机械与结构

关键要问：
- 这个母题能不能换一个载体继续出现？
- 如果只是原样再来一次，会不会显得重复？

### 4. 明确哪些东西不能直接复用

一定要单独写下来：哪些东西不能再用同一种形式重演。

例如：
- 同一种医院逻辑
- 同一种旧病人/旧案回潮
- 同一种物件驱动异常
- 同一种情绪负担只是提高强度

## 推荐输出

在正式提出候选 arc 之前，先写一份很短的内部清单：
- keep（继续保留）
- transform（换形态继续用）
- avoid repeating（禁止原样重复）
- exploit next（下一篇最该利用）

这样能有效避免拍脑袋想点子。
