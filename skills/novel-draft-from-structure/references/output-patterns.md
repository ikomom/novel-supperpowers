# 按结构写正文的输出模式

## 模式 A：单章写作包

```markdown
# <章号 + 标题>

## 本章功能
- 

## 本章必须出现
- 
- 
- 

## 本章必须避免
- 
- 

## 章尾要落到
- 
```

然后再写完整正文。

## 模式 B：多章批量写作

批量写几章时，每章动笔前都先明确：
- 功能
- 核心事件
- 揭示层级
- 章尾转折

不要把三章连写成一团模糊的连续文本。
每章都必须有自己的身份。

## 模式 C：把抽象 beat 换成场景 beat

如果提纲太抽象，可以这样转换：

- 抽象 beat：主角意识到异常想要一个名字
- 场景 beat：主角在标签、记录或屏幕上看见名字自己冒出来，并且必须立刻做出动作反应

优先写场景 beat，不要直接写抽象判断。
