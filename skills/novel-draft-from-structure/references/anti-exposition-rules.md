# 防站桩解释规则

按结构写正文时，优先用这套规则防止小说发虚。

## 1. 不要停下小说来解释小说

如果两个角色本来就知道危险，不要让他们停下来长篇复述，只为了把设定讲给读者听。

更好的做法：
- 在移动中解释
- 在解释时被新证据打断
- 让解释直接改变下一步行动

## 2. 解释必须赚到位置

一段解释至少要做到其中之一，才值得留下：
- 改变主角下一步动作
- 露出新的代价
- 制造角色冲突
- 重写目标或问题本身

如果它只是重复已知信息，就删掉。

## 3. 多用物件和程序

涉及高概念时，尽量把它们落在这些东西上：
- 纸条
- 标签
- 档案
- 工具
- 身体症状
- 房间布局
- 操作流程

## 4. 保护主角

不要让主角连续几页只负责听别人讲话。

主角应该：
- 试探
- 打断
- 拒绝错误框架
- 做临时判断
- 在不完全确定时先行动

## 5. 章尾要落在动作，不要落在总结

尽量避免章节结尾只是把刚才学到的东西总结一遍。

更好的章尾通常是：
- 有东西来了
- 有东西坏了
- 局面翻了一下
- 下一个目标被点出来
- 读者知道必须立刻继续往下看
